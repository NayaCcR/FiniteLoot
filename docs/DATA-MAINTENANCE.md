# 数据、备份与升级

数据库固定为 `plugins/FiniteLoot/data.db`，包含容器、玩家领取、个人库存和 schema 版本数据。个人奖励和剩余物品会在关闭 GUI、玩家退出及插件关闭时保存。

## 升级

1. 运行 `/fl migrate`。
2. 确认 `plugins/FiniteLoot/backups` 中生成带时间戳的备份。
3. 停止服务器并备份整个 `plugins/FiniteLoot` 目录。
4. 替换 JAR 后启动服务器。

迁移只向前升级，不主动删除旧版本数据。不要在服务器运行时直接复制 `data.db`，SQLite WAL 可能仍有已提交内容。

## 故障行为

数据库故障时，插件拒绝新的领取资格，不会回退为公共箱或无限生成。孤立数据清理只通过 `/fl migrate` 执行，并在清理前创建备份。

容器使用 PDC 中的 UUID 作为主键，双箱会归一为同一个容器 ID；世界或区块重新生成不会因为坐标相同而继承旧记录。
